# <类型>(<范围>): <主题>
# 例如: feat(user): 新增用户登录功能
# -------------------------
# 可选类型:
#   feat:     新功能
#   fix:      修复 Bug
#   docs:     文档变更
#   style:    代码格式化
#   refactor: 重构代码
#   test:     测试用例
#   chore:    构建/工具变更
# -------------------------
# 主题使用现在时，首字母小写，结尾不加句号
# 正文（可选）详细描述变更内容
# -------------------------
# 底部可添加关联的 Issue（如 fixes #123）
# -------------------------
